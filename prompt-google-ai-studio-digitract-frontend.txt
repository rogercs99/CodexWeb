PROBLEMAS QUE DEBES CORREGIR
1. App monolitica (actualmente un archivo gigante) -> modularizar por dominios.
2. Flujo lobby/start inconsistente -> separar claramente start clasico vs start_with_bots.
3. Temporizadores visuales no confiables -> countdown reactivo, preciso y estable.
4. Estados UX incompletos -> loading/empty/error/success en todas las vistas.
5. Feedback inconsistente -> toasts/alerts/modales estandarizados y no bloqueantes.
6. Accesibilidad pobre -> semantica, focus, teclado, aria, contraste, alt, focus-trap.
7. Responsive incompleto -> experiencia cuidada en movil y desktop.
8. Dependencias remotas en runtime (CDN/fuentes/fondos) -> empaquetado local y reproducible.
9. Deuda de lint/types -> codigo limpio sin warnings importantes.
10. Elementos legacy/dead code -> aislar o eliminar sin romper funcionamiento.

REQUISITOS DE ARQUITECTURA
